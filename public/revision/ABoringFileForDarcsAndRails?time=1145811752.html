  1. 0. h1. A boring file for darcs and rails
  2. 1.
  3. 2. Rails has a fair number of files that it isn't useful to include under version control. To manage this with darcs I create a file called boring with the content below, and then issue @darcs setpref boringfile boring@.
  4. 3.
  5. 4. You can then use @darcs add -r *@ to add everything else to darcs, and everytime you add new files.
